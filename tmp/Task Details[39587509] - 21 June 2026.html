These were a particularly challenging task in BC. It includes the following:

  • Content management: since many products share the same lists, perhaps a metafield to select a collection would best fit. This way the client could create and manage each list as a collection and be confident if they updates one collection then all the products using that collection would be updated.

  • Product page presentation: This exists in 3 states:

    • Pre-selection state: When you first land on the page. The dropdown is collapsed and contains the words "Choose Options:

    • The selection state: Dropdown is open and contains a scrollable list of the products

    • The selected state: Following a selection, the drop down is collapsed and now contains the selected product with a change button

  • Cart (and cart drawer) presentation: The selected container is grouped with the primary product in the shopping cart row. I believe Shopify has a native 'product grouping’ option available to support this. The customer should have a link in the cart which will allow them to change the container selection.

  • Checkout presentation: Like the cart, the selected container is grouped with the primary product in the shopping cart row

  • Order fulfillment: The order itself should clearly identify that the selected container belongs with the primary product so fulfillment knows how to ‘build’ the product before shipping it.

This may be solvable with an OTS app, or may require custom development. You should identify if you can do this with an app within the first hour of task time.
